AI 面试陪练系统学习地图

这张图只解决一个问题：当你看代码看到后面忘了前面时，快速定位“我现在在哪一层、这段代码负责什么、下一步会流向哪里”。

先背这条主线

  1. CLI 接收输入
  2. InterviewerAgent 组织角色和工具
  3. AgentLoop 反复调用 LLM
  4. ToolRegistry 执行工具
  5. Tools 读题库或写数据库
  6. LLM 组织最终回复
口诀：CLI 管输入输出，Agent 管角色，Loop 管循环，Registry 管工具，Tools 管业务，DB 管记忆。

一条完整执行链路

从用户输入到面试官回复
用户输入回答题目、要求换题、查看统计或退出。
cli_interview.py命令行外壳，负责 input 和 print。
InterviewerAgent面试官角色，持有工具、状态和 AgentLoop。
AgentLoop.run把消息、prompt、tools 发给 LLM。
LLM 判断决定直接回复，还是发起 tool_call。
ToolRegistry解析 tool_call，执行对应 Python 函数。
业务工具question_tools 读题库，memory_tools 写数据库。
最终回复工具结果回到 LLM，生成面试官文本。

系统分层地图

中间是心脏，左右是入口和工具

入口层

  • scripts/cli_interview.py
  • 打印欢迎页
  • 创建 InterviewerAgent
  • 读取用户输入
  • 处理 help、stats、quit

角色层

  • agents/roles/interviewer_agent.py
  • 继承 Agent
  • 注册 4 个工具
  • 保存 current_question
  • 保存 session_id

Agent 基类层

  • agents/core/base_agent.py
  • 读取 harness.yaml
  • 加载 interviewer.md
  • 创建 LLM 客户端

AgentLoop.run()

一次用户输入可能触发多轮 LLM，因为 LLM 可能先调用工具，再根据工具结果继续思考。

Think构造 messages 和 tools，调用 LLM。
Act发现 LLM 返回了 tool_calls。
Observe执行工具，把结果写回 messages。
Reflect没有工具调用时，返回最终文本。

工具注册层

  • agents/core/tool_registry.py
  • 注册 Python 函数
  • 生成 OpenAI tools schema
  • 解析 tool_call
  • 执行对应函数

业务工具层

  • question_tools.py 读取题库
  • memory_tools.py 读写数据库
  • 抽题、查模块、查薄弱点
  • 保存评分、计算复习时间

数据层

  • 知识库/Java面试
  • .harness/db/learning.db
  • agents/definitions/interviewer.md
  • 题目、学习记录、人设规则

两条最重要路径

启动一次面试，以及回答一道题

第一次启动面试

cli_interview.py 创建 InterviewerAgent()
base_agent.py 加载配置和面试官 prompt
interviewer_agent.py 注册工具，创建 AgentLoop
start_interview() 创建 session_id
AgentLoop.run() 调 LLM，可能查薄弱模块并抽题
返回第一道面试题，CLI 打印给你

用户回答一道题

CLI 通过 input() 拿到你的回答
调用 continue_interview(user_input)
AgentLoop.run() 把回答放入 messages
LLM 判断这是答案，可能调用 save_evaluation
memory_tools 保存学习记录并安排复习
LLM 给出反馈或追问，CLI 打印出来

六张回忆卡片

每天复习 3 分钟
卡片 1

CLI 负责什么？负责命令行输入输出，普通输入交给 Agent。

卡片 2

InterviewerAgent 负责什么？组装面试官，注册工具，维护当前题目和会话。

卡片 3

AgentLoop 为什么循环？因为 LLM 可能先调工具，再根据结果继续思考。

卡片 4

LLM 会直接执行工具吗？不会。它只发 tool_call，ToolRegistry 才真正执行。

卡片 5

题目从哪里来？从 知识库/Java面试 的 Markdown 文件来。

卡片 6

学习记录存哪里？存在 .harness/db/learning.db 的表里。
